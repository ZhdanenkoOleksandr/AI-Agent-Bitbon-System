# Структура проекта ACADEMY PLATFORM ECONOMY

Полное описание структуры проекта и содержимого каждой папки.

## 📁 Основная структура

```
src/academy-platform-economy/
├── agents/                                 # Многоагентная система
│   ├── transcriber/                       # ✓ Transcriber Agent (готов)
│   │   ├── config/
│   │   │   ├── agent-config.json          # Конфигурация агента
│   │   │   └── system-prompt.txt          # Системные инструкции
│   │   └── README.md                      # Документация агента
│   │
│   ├── agent-template/                    # Шаблон для новых агентов
│   │   ├── config/
│   │   │   ├── agent-config-template.json # Шаблон конфига
│   │   │   └── system-prompt-template.txt # Шаблон промпта
│   │   └── README-TEMPLATE.md             # Шаблон документации
│   │
│   ├── agents-registry.json               # Реестр всех агентов
│   └── [другие агенты будут здесь]
│
├── content/                               # Контент платформы
│   ├── lectures/                          # Готовые лекции
│   ├── videos/                            # Исходные видеофайлы
│   ├── audio/                             # Исходные аудиофайлы
│   └── drafts/                            # Черновики
│
├── workflows/                             # Рабочие процессы
│   ├── transcription-flow.json            # Поток транскрипции (планируется)
│   └── publishing-flow.json               # Поток публикации (планируется)
│
├── config/                                # Конфигурация проекта
│   ├── project-config.json                # Основная конфигурация
│   ├── swarm-topology.json                # Топология многоагентной системы
│   └── standards.json                     # Стандарты качества (планируется)
│
├── README.md                              # Главная документация
├── PROJECT.md                             # Подробное описание проекта
├── AGENTS_GUIDE.md                        # Руководство по агентам
└── STRUCTURE.md                           # Этот файл
```

## 📄 Описание файлов

### Агенты (agents/)

#### transcriber/
**Статус**: ✓ Готов к использованию

- `config/agent-config.json` — Полная конфигурация агента с входами/выходами
- `config/system-prompt.txt` — Детальные инструкции для агента
- `README.md` — Документация с примерами использования

**Возможности**:
- Преобразование аудио в текст
- Преобразование видео в текст
- Подтверждение готового текста
- Сохранение структуры речи

#### agent-template/
**Назначение**: Шаблон для создания новых агентов

- `config/agent-config-template.json` — Заготовка конфигурации
- `config/system-prompt-template.txt` — Заготовка промпта
- `README-TEMPLATE.md` — Заготовка документации

**Использование**: Скопируйте файлы, замените [REPLACE_*] на свои значения

#### agents-registry.json
**Назначение**: Центральный реестр всех агентов

**Содержит**:
- Активные агенты (с заполненной информацией)
- Планируемые агенты (со статусом "planned")
- Конфигурацию топологии системы
- Стандарты качества

**Обновляйте** при добавлении новых агентов!

### Контент (content/)

#### content/lectures/
**Назначение**: Готовые к использованию лекции

Формат файла:
```markdown
# [Название лекции]

## Метаданные
- Спикер: [Имя]
- Дата: [Дата]
- Язык: [Язык]

## Содержание

[Полный текст лекции]
```

#### content/videos/
**Назначение**: Исходные видеофайлы для транскрипции

Поддерживаемые форматы: mp4, webm, mov, avi

#### content/audio/
**Назначение**: Исходные аудиофайлы для транскрипции

Поддерживаемые форматы: mp3, wav, m4a, ogg

#### content/drafts/
**Назначение**: Черновики, обрабатываемые контент

Может содержать частично готовый контент, промежуточные версии

### Рабочие процессы (workflows/)

#### transcription-flow.json (планируется)
**Назначение**: Определение процесса транскрипции

Будет содержать:
- Шаги обработки
- Распределение между агентами
- Обработка ошибок
- Callback'и

#### publishing-flow.json (планируется)
**Назначение**: Определение процесса публикации

Будет содержать:
- Валидация контента
- Публикацию на платформу
- Распределение по каналам

### Конфигурация (config/)

#### project-config.json
**Назначение**: Основная конфигурация проекта

**Содержит**:
- Версию и описание проекта
- Параметры агентов (модель, температура, timeout)
- Конфигурацию памяти и хранилища
- Параметры безопасности и качества
- Интеграции и функции

#### swarm-topology.json
**Назначение**: Описание топологии многоагентной системы

**Содержит**:
- Иерархию слоев (Orchestration, Processing, Analysis, Distribution)
- Определение агентов по ролям
- Параметры консенсуса (Raft)
- Стратегии масштабирования
- Определение рабочих процессов

#### standards.json (планируется)
**Назначение**: Стандарты качества и требования к контенту

Будет содержать:
- Требования к транскрипции
- Стандарты форматирования
- Минимальные уровни точности
- Чек-листы для QA

## 🔄 Рекомендуемые действия при работе с проектом

### Добавление нового агента

1. Прочитайте [AGENTS_GUIDE.md](AGENTS_GUIDE.md)
2. Создайте папку: `agents/[agent-name]/`
3. Скопируйте файлы из `agents/agent-template/`
4. Адаптируйте конфигурацию
5. Создайте документацию README.md
6. Добавьте агента в `agents/agents-registry.json`
7. Обновите `config/swarm-topology.json`

### Добавление новой лекции

1. Получите исходное видео/аудио
2. Поместите в `content/videos/` или `content/audio/`
3. Создайте задачу для Transcriber Agent
4. Получите готовый текст
5. Сохраните в `content/lectures/` с метаданными
6. Опционально: пройдите через Editor, QA агентов

### Обновление конфигурации

1. Отредактируйте нужный файл в `config/`
2. Перезагрузите конфигурацию: `npx @claude-flow/cli@latest config reload`
3. Проверьте здоровье системы: `npx @claude-flow/cli@latest doctor --fix`

## 🗂️ Соглашения об именовании

### Папки
- Используйте kebab-case: `agent-name`, `content-type`
- Групповые папки в нижнем регистре: `agents`, `content`, `config`

### Файлы конфигурации
- JSON: `*-config.json`, `*-registry.json`
- Текст: `system-prompt.txt`, `instructions.txt`
- Markdown: `README.md`, `GUIDE.md`

### Папки агентов
```
agents/[agent-name]/
├── config/
│   ├── agent-config.json
│   └── system-prompt.txt
└── README.md
```
