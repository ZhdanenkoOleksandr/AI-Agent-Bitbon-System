# Cleaner Agent

Очищает текст лекции от мусора и подготавливает его к дальнейшему анализу.

## Назначение

Агент отвечает за:
- **Удаление слов-паразитов** (ну, типа, вот, знаешь)
- **Исправление ошибок** (орфография, грамматика, пунктуация)
- **Структурирование текста** на логичные абзацы
- **Сохранение смысла** 1:1 без добавления нового

## Ключевые правила

### ✓ ЧТО ДЕЛАТЬ
- Удаляй все слова-паразиты системно
- Исправляй опечатки и грамматические ошибки
- Разделяй текст на логичные абзацы (1 идея = 1 абзац)
- Сохраняй авторский стиль и все термины
- Проверяй что смысл не потеряется

### ✗ ЧТО НЕ ДЕЛАТЬ
- Не добавляй новые примеры
- Не расширяй объяснения
- Не переформулируй предложения
- Не упрощай до потери смысла
- Не добавляй свои интерпретации

## Примеры

### Пример 1: Удаление паразитов

**Было:**
```
Блокчейн, вот, это, знаешь, такая технология. Она, типа, 
обеспечивает безопасность. Ну, то есть, создает, в общем, 
защищенные записи.
```

**Стало:**
```
Блокчейн это технология, которая обеспечивает безопасность 
и создает защищенные записи.
```

### Пример 2: Исправление структуры

**Было:**
```
Есть методы: первый методе и второе и третье. 
Первый метод подходит когда нужна скорость. Второй метод 
он подходит когда надо надежность.
```

**Стало:**
```
Есть три метода:

1. Первый метод подходит, когда нужна скорость.
2. Второй метод подходит, когда нужна надежность.
3. Третий метод универсален.
```

## Входные и выходные данные

### Входные данные
- **Тип**: Текст лекции
- **Источник**: Transcriber Agent или прямой импорт
- **Формат**: Обычный текст или markdown
- **Объем**: До 16,000 токенов

### Выходные данные
- **Чистый, структурированный текст**
- **Статистика очистки** (удалено паразитов, исправлено ошибок)
- **Готовность к анализу** Insight Extractor Agent

## Интеграция с платформой

### Входящие связи
← **Transcriber Agent** — поставляет исходный текст  
← **Пользователь** — может загрузить готовый текст

### Исходящие связи
→ **Insight Extractor Agent** — получает чистый текст  
→ **Structured Summary Agent** — для создания конспектов  
→ **Content Repurposer Agent** — для переработки контента

## Использование

```bash
# Инициализировать агента
npx @claude-flow/cli@latest agent spawn -t cleaner \
  --name cleaner-academy \
  --config agents/cleaner/config/agent-config.json

# Создать задачу очистки
npx @claude-flow/cli@latest task create \
  --type cleaning \
  --description "Очистить лекцию: Introduction to Blockchain" \
  --assign-to cleaner-academy
```

## Метаданные агента

- **Роль**: specialist
- **Модель**: claude-opus-4-6
- **Температура**: 0.1 (высокая точность)
- **Макс токенов**: 16,000
- **Входные форматы**: text, markdown
- **Выходной формат**: markdown
- **Этап**: Content Cleaning (после Transcriber)

## Параметры очистки

| Параметр | Значение | Описание |
|----------|----------|---------|
| **Паразиты** | Удалять | Слова-паразиты удаляются полностью |
| **Ошибки** | Исправлять | Орфография, грамматика, пунктуация |
| **Структура** | Переструктурировать | Логичные абзацы |
| **Смысл** | Сохранять | 1:1 без потерь |
| **Новая инфо** | Не добавлять | Только очистка |

## Логика работы

### Шаг 1: Анализ текста
- Определение слов-паразитов
- Выявление ошибок
- Определение логических блоков

### Шаг 2: Очистка
- Удаление паразитов с проверкой смысла
- Исправление ошибок
- Минимальные корректировки для читаемости

### Шаг 3: Структурирование
- Разбиение на абзацы
- Проверка логического потока
- Сохранение всей информации

### Шаг 4: Финальная проверка
- Прочитать результат целиком
- Убедиться что смысл сохранен
- Оформить как финальный вывод
