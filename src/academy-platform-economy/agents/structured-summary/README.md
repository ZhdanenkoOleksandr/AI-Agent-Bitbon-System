# Structured Summary Agent

Создает понятный, логичный конспект лекции в 5 секциях для чтения и изучения.

## Назначение

Агент отвечает за:
- **Создание структурированного конспекта** с 5 логичными секциями
- **Упрощение сложных идей** простым языком
- **Организацию информации** от простого к сложному
- **Готовность к применению** (знания, готовые к использованию)

## Ключевые правила

### ✓ ЧТО ДЕЛАТЬ
- Пиши простым языком для новичка
- Структурируй информацию логично
- Включай примеры для ясности
- Сохраняй все ключевые знания
- Делай вывод понятным и практичным

### ✗ ЧТО НЕ ДЕЛАТЬ
- Не усложняй описание
- Не пропускай важные моменты
- Не добавляй информацию вне лекции
- Не используй сложный жаргон без объяснения
- Не делай конспект скучным

## Структура конспекта

1. **О чем эта лекция** (введение)
2. **Основные идеи** (главные мысли кратко)
3. **Подробное объяснение** (расширенное изложение)
4. **Примеры** (практические случаи)
5. **Выводы** (итоги и применение)

## Примеры

### Пример конспекта

```markdown
# Основы личного брендинга

## О чем эта лекция
Эта лекция о том, как создать мощный личный бренд, 
который откроет вам двери в карьере и бизнесе.

## Основные идеи
1. Личный бренд = ваша репутация в интернете
2. Контент привлекает людей
3. Сообщество умножает влияние
4. Продукт монетизирует ваш бренд

## Подробное объяснение
Личный бренд строится на четырех столпах...
[подробное объяснение каждой идеи]

## Примеры
Пример 1: Как Илон Маск использует Twitter...
Пример 2: Как блогер построил аудиторию за 2 года...

## Выводы
Главное: начните создавать контент уже сегодня...
```

## Входные и выходные данные

### Входные данные
- **Тип**: Insights + Clean Text
- **Источник**: Insight Extractor Agent + Cleaner Agent
- **Формат**: Structured analysis + raw text
- **Объем**: До 16,000 токенов

### Выходные данные
- **Структурированный конспект** (5 секций)
- **Markdown формат** (готов к публикации)
- **Понятный язык** (для новичка)
- **Практичное содержимое** (применимо)

## Интеграция с платформой

### Входящие связи
← **Insight Extractor Agent** — анализ содержимого  
← **Cleaner Agent** — чистый текст  

### Исходящие связи
→ **Lesson Builder Agent** — для создания полного урока  
→ **Personalization Agent** — для адаптации  
→ **Content Repurposer Agent** — для социального контента  

## Использование

```bash
# Инициализировать агента
npx @claude-flow/cli@latest agent spawn -t summary \
  --name summary-academy \
  --config agents/structured-summary/config/agent-config.json

# Создать задачу конспекта
npx @claude-flow/cli@latest task create \
  --type summarization \
  --description "Создать конспект лекции: [название]" \
  --assign-to summary-academy
```

## Метаданные агента

- **Роль**: specialist
- **Модель**: claude-opus-4-6
- **Температура**: 0.2 (высокая точность)
- **Макс токенов**: 16,000
- **Входной формат**: structured analysis + markdown
- **Выходной формат**: markdown
- **Этап**: Content Summarization (после Insight Extraction)

## Параметры конспекта

| Параметр | Значение | Описание |
|----------|----------|---------|
| **Секции** | 5 | Стандартная структура |
| **Язык** | Простой | Для новичка |
| **Примеры** | 2-3 | Для ясности |
| **Объем** | 800-1500 слов | Сжато но полно |
| **Формат** | Markdown | Готов к публикации |

## Логика работы

### Шаг 1: Анализ входных данных
- Прочитать insights и clean text
- Понять основную структуру
- Выделить ключевые блоки

### Шаг 2: Написание секции 1 (О чем)
- Краткое резюме: в чем суть лекции
- 2-3 предложения для ясности

### Шаг 3: Написание секции 2 (Основные идеи)
- Список 5-7 главных идей
- Краткие описания (1-2 строки)

### Шаг 4: Написание секции 3 (Подробное объяснение)
- Развернутое изложение каждой идеи
- Логичные переходы
- Примеры встроены

### Шаг 5: Написание секции 4 (Примеры)
- 2-3 практических примера
- Из реальной жизни если возможно
- Связь с основной идеей

### Шаг 6: Написание секции 5 (Выводы)
- Итоги лекции
- Как применить на практике
- Подготовка к следующему шагу
