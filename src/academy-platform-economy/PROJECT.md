# ACADEMY PLATFORM ECONOMY

Многоагентная образовательная платформа для создания, обработки и публикации лекционного контента.

## Обзор

Проект использует специализированных агентов для автоматизации полного цикла работы с контентом:

```
RAW CONTENT → [Transcriber] → [Editor] → [Translator] → [Summarizer] → [QA] → [Publisher] → PUBLISHED
     ↑                                                                                         ↓
     └─────────────────── [Feedback Loop] ──────────────────────────────────────────────────┘
```

## Быстрый старт

### 1. Инициализация проекта

```bash
cd src/academy-platform-economy

# Посмотреть реестр агентов
cat agents/agents-registry.json

# Посмотреть руководство по агентам
cat AGENTS_GUIDE.md
```

### 2. Запуск Transcriber Agent

```bash
# Инициализировать агента
npx @claude-flow/cli@latest agent spawn \
  -t transcriber \
  --name transcriber-academy \
  --config agents/transcriber/config/agent-config.json

# Создать задачу транскрипции
npx @claude-flow/cli@latest task create \
  --type transcription \
  --description "Транскрибировать лекцию: Introduction to Blockchain" \
  --assign-to transcriber-academy
```

### 3. Инициализировать свармом

```bash
# Инициализировать многоагентную систему
npx @claude-flow/cli@latest swarm init \
  --topology hierarchical-mesh \
  --max-agents 15 \
  --strategy specialized
```

## Активные агенты

| Агент | Статус | Задача | Входы | Выходы |
|-------|--------|--------|-------|--------|
| **Transcriber** | ✓ Готов | Преобразование аудио/видео в текст | mp3, wav, mp4, webm | Текст, метаданные |

## Планируемые агенты

| Агент | Статус | Задача | Порядок |
|-------|--------|--------|--------|
| Editor | Запланирован | Редактирование и улучшение | 2 |
| Translator | Запланирован | Перевод контента | 3 |
| Summarizer | Запланирован | Создание резюме | 4 |
| QA | Запланирован | Проверка качества | 5 |
| Publisher | Запланирован | Публикация | 6 |

## Best Practices

### ✓ Что делать

1. **Документируйте все изменения**
   - Обновляйте README для каждого агента
   - Регистрируйте новых агентов в реестре
   - Описывайте входы и выходы

2. **Следуйте стандартам качества**
   - Используйте температуру 0 для точности
   - Валидируйте все входные данные
   - Стандартизируйте форматы вывода

3. **Тестируйте интеграцию**
   - Проверяйте совместимость между агентами
   - Используйте пример контента
   - Отслеживайте метаданные

4. **Ведите логи**
   - Записывайте выполнение каждой задачи
   - Отслеживайте ошибки
   - Анализируйте производительность

### ✗ Что не делать

- ❌ Не смешивайте ответственности в одном агенте
- ❌ Не меняйте температуру без причины
- ❌ Не игнорируйте валидацию входных данных
- ❌ Не забывайте документировать изменения

## Добавление нового агента

1. Создайте папку: `agents/[agent-name]/`
2. Скопируйте шаблоны из `agents/agent-template/`
3. Адаптируйте конфигурацию под свои нужды
4. Создайте документацию (README.md)
5. Зарегистрируйте в `agents/agents-registry.json`
6. Протестируйте интеграцию

Подробное руководство: см. [AGENTS_GUIDE.md](AGENTS_GUIDE.md)
