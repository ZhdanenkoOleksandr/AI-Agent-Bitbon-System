# Testing Telegram Login Flow

## Prerequisites

1. Server running: `npm start`
2. A partner with status "active" in the database
3. Access to browser DevTools

## Test Steps

### Step 1: Verify Backend

Check that the server has the required data structures:

```bash
curl http://localhost:3000/api/status
# Should return 200 OK
```

### Step 2: Create Test Partner (if needed)

```bash
# The partner must exist in DB.partners with:
# - status: "active"
# - telegram: "@username"
# - firstName, lastName, email, etc.
```

### Step 3: Generate Web Token

Manually trigger the bot's /start handler:

```bash
# This would normally happen when user clicks Telegram button
# For testing, you can manually create a web_token
```

### Step 4: Test Token Exchange

Use curl to test the API endpoint:

```bash
# First, get a valid web_token from the database
# Then exchange it:

curl "http://localhost:3000/api/auth/web-token?wt=abc123def456..."

# Expected response:
# {
#   "success": true,
#   "jwt": "eyJhbGciOiJIUzI1NiIs...",
#   "user": {
#     "id": "prt_xxx",
#     "fullName": "John Doe",
#     "email": "john@example.com",
#     "telegram": "@johndoe",
#     "status": "active",
#     "packageType": "pro",
#     ...
#   }
# }
```

### Step 5: Test Full Flow in Browser

1. **Open DevTools** (F12)
2. **Go to Application tab → Storage → localStorage**
3. **Clear all localStorage** (optional, for clean test)
4. **Navigate to:**
   ```
   http://localhost:3000/?wt=TOKEN&username=@testuser
   ```
5. **Watch the following happen:**
   - Network tab shows `GET /api/auth/web-token?wt=...`
   - localStorage gets populated:
     - `jwt: eyJ...` (JWT session token)
     - `user: {"id":"prt_xxx",...}` (user object)
   - URL changes from `/?wt=TOKEN` to `/cabinet`
   - Page redirects to cabinet

### Step 6: Verify Session Restoration

1. **Refresh the page** (Cmd+R / Ctrl+F5)
2. **Check DevTools → Application → localStorage**
   - JWT should still be there
   - User data should still be there
3. **The cabinet should load** with user's name/status visible

## Testing Checklist

- [ ] Web token generated by bot (10-minute TTL)
- [ ] Token exchange returns JWT
- [ ] JWT stored in localStorage
- [ ] User data stored in localStorage
- [ ] URL cleaned (history.replaceState)
- [ ] Redirects to /cabinet
- [ ] Session persists on page refresh
- [ ] User mode switches to 'partner'
- [ ] API calls include Authorization header with JWT

## Common Issues

### Issue: "Token not found or expired"
**Solution:** Make sure the web_token exists in `DB.webTokens` and hasn't expired

### Issue: "Partner not found"
**Solution:** Verify the partner exists with the correct Telegram username

### Issue: JWT not in localStorage
**Solution:** Check browser's localStorage settings (not cleared, not disabled)

### Issue: Redirect to /cabinet doesn't work
**Solution:** Check if window.location.href is allowed by browser
